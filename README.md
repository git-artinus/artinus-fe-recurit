# ARTINUS 프론트엔드 과제 전형

---

## 개요

여러 서비스의 회원가입 페이지를 하나의 코드베이스에서 개발하고 운영하는 과제입니다.

서비스별 요구사항 차이를 공통 구조 안에서 효율적으로 관리할 수 있도록 설계해 주세요. 새로운 서비스가 추가되더라도 최소한의 코드 수정으로 대응할 수 있어야 합니다.

구현 내용에는 재사용 가능한 페이지 구조 설계, 가입 플로우의 상태 관리, Mock API 연동·오류 대응, 서비스별 독립 빌드 등을 포함합니다.

---

## 요구사항

아래 세 서비스의 회원가입 페이지를 구현하세요.

### 회원가입 구성

서비스마다 입력 항목·순서·검증·약관이 다릅니다. 구체적인 문구·약관 내용은 자유롭게 정하셔도 됩니다.

| 항목    | 커뮤니티 서비스 | 뉴스 구독 서비스 | 쇼핑 멤버십 서비스                                                      |
|-------|----------------|----------------|-----------------------------------------------------------------|
| 공통 항목 | 페이지 타이틀(text), 소개 배너(image) | 페이지 타이틀(text), 소개 배너(image) | 페이지 타이틀(text), 소개 배너(image)                                     |
| 입력 항목 | ID, 비밀번호, 비밀번호 확인, 휴대폰 인증 | 휴대폰 인증, 비밀번호, 비밀번호 확인 | ID, 비밀번호, 비밀번호 확인, 생년월일, 휴대폰 인증                                 |
| 약관 | (필수) 서비스 이용약관, (필수) 개인정보 수집 및 이용 | (필수) 서비스 이용약관, (필수) 개인정보 수집 및 이용, (선택) 마케팅 수신 동의 | (필수) 서비스 이용약관, (필수) 개인정보 수집 및 이용, (필수) 만 14세 이상, (선택) 마케팅 수신 동의 |
| 테마 | 보라 계열 | 초록 계열 | 주황 계열                                                           |

### 공통 동작 요건
- 타이틀·배너: 서비스마다 다르게 노출 (문구·이미지 자유)
- input 항목에 validation 적용 (기준 자유)
- 휴대폰 인증
  - 인증번호 받기 → 입력 → 검증. 3분 카운트다운
  - 인증번호 발송 API는 구현하지 않습니다. '인증번호 받기' 버튼 클릭 시 클라이언트에서 타이머를 시작하고 인증번호 입력 상태로 진입하면 됩니다.
    - 인증 만료(타이머 종료) 여부는 클라이언트 타이머로 판단합니다. 만료 시 인증 요청 불가, '인증번호 받기' 클릭 후 다시 시도 가능
  - 인증번호 검증은 [Mock API 구현](#mock-api-구현)에 따라 구현합니다.
  - 인증 상태(대기, 성공, 실패 등)를 구분하여 처리해 주세요.
  - 검증 요청 중에는 로딩 상태를 표시하고, 중복 요청을 방지해 주세요.
- 약관 동의: 필수/선택 구분, 전체 ↔ 개별 동의 동기화, 필수 약관 미동의 시 가입 불가
- 가입하기: 모든 필수 입력이 유효하고, 휴대폰 인증 및 필수 약관 동의가 완료된 경우에만 활성화 (실제 가입 처리는 구현 제외)
- 반응형: 모바일·데스크톱 모두 대응
- 디자인·UI·UX 자유롭게 구현

### 확장성
- 위 세 서비스 외에도 서비스는 계속 추가되고 항목·순서·문구 변경도 잦습니다. 이를 고려한 설계를 해 주세요.
- 예시: 신규 서비스 추가, 입력 항목 추가, 순서 변경, 약관 종류 추가 등


### 빌드
- 라우팅 기반 분리가 아닌, 서비스별로 독립된 HTML 산출물이 생성되어야 합니다.
- 예시: `dist/community.html`, `dist/news.html`, `dist/shopping.html`


## Mock API 구현

아래 API 계약을 기준으로 프론트엔드 내에서 Mock API를 구현해 주세요.
구현 방식(MSW, Mock Service, fetch wrapper 등)은 자유입니다.

### 휴대폰 인증번호 검증 
> POST /api/verify

Mock API는 아래 규칙으로 동작합니다.
- code === "123456" → 인증 성공
- code === "999999" → 일시적 오류
- 그 외 → 인증 실패
- 응답은 0.5~1.5초 랜덤 지연 후 반환되도록 구현해 주세요.

- 예시
    ```jsonc 
    // Request
    { "mobile": "01012345678", "code": "123456" }
    
    // Response
    // 인증 성공 - HTTP 200
    { "verified": true }
  
    // 인증 실패 - HTTP 200 (코드 불일치) → 코드 재확인 안내
    { "verified": false, "reason": "MISMATCH" }
  
    // 일시적 오류 - HTTP 500 → "다시 시도" 안내
    { "error": "INTERNAL" }
    ```
